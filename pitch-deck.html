02 / 09
The Problem

Crypto payments are inconvenient.
Nobody wants to scan QR codes.

Terrible UX

QR codes, long wallet addresses, copy-paste flows. Merchants can't use this at a point-of-sale. Customers give up at checkout.

Slow Settlement

Card payments take 1–3 days to clear. Even crypto solutions add layers of off-chain complexity that slow everything down.

High Fees

Credit card processors charge 1.5–3% per transaction. Chargebacks cost merchants over $100B/year globally. That's high.

03 / 09
Our Insight

Poland already solved this.
It's called BLIK.

BLIK is a mobile payment system used by 20M+ Poles. A 6-digit code, 2-minute expiry, bank-app confirmation. No card numbers. No QR codes. Just a code you say out loud.

  • 2.9 billion transactions in 2025
  • 58% of Poles aged 15+ use BLIK monthly
  • 50%+ of all Polish e-commerce payments
  • Generated 1.2% of Poland's GDP (~€9.5B)
  • Supported by 90% of Polish banks

BLIK's Proven Model

01
Generate – 6-digit code in banking app
02
Enter – Type code at merchant checkout
03
Confirm – Approve in app (amount + merchant)
04
Done – Instant settlement, receipt generated

We replicate this exact flow – on Solana.

04 / 09
The Solution

SLIK = BLIK + Solana

Same proven 6-digit code flow. But instead of banks, it's blockchain. Instant, atomic, irreversible.

Merchant enters amount

PLN / USD / EUR → auto-converts to SOL at live rate or use stablecoin

Customer generates code

Random 6-digit code, cryptographically secure, 120s TTL

Code links to payment

Merchant enters code, system matches it atomically via Redis

Approval & settlement

One wallet tap. SOL / Stable transferred + on-chain receipt in ~400ms
